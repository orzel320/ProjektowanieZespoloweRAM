### 1. Executive Summary
The primary objective of Sprint 1 was to establish the foundational machine learning pipeline capable of translating words generated by the AI service into mathematical vector representations. We successfully built a dynamic, caching-based embedding system that operates seamlessly with the backend's generative Pydantic models.

### 2. Key Architectural Decisions

* **Language & Model Selection:** Decided to restrict the initial scope to the English language. This allowed the utilisation of the `all-MiniLM-L6-v2` model via the `sentence-transformers` library. It is exceptionally fast, lightweight, and highly optimised for English semantic mapping.
* **Dynamic Caching vs. Static Dictionary:** Initially, the plan involved generating a massive static JSON dictionary. However, after reviewing the AI Engineer's integration (which dynamically generates obscure vocabulary, especially on 'Expert' difficulty), we pivoted to an **on-the-fly embedding mechanism**. 
* **The Repository Pattern:** Direct database dependencies were avoided. We implemented an abstract `VectorStore` interface with a concrete `JsonVectorStore` class. This ensures that future migration to a dedicated vector database (like Qdrant or PostgreSQL with pgvector) will require zero modifications to the core scoring or API logic.

### 4. Key Discoveries & The "Wordplay Trap"

During testing against official NYT puzzles, a critical observation was made regarding AI constraints. We analysed a group relying on wordplay rather than semantics (*SODA, STORMY, TAN, TONIC* — all forming words when combined with external context). 

* **Observation:** The ML model evaluates these words with a near-zero semantic similarity, as it cannot read the "hidden" context.
* **Actionable Pivot for Sprint 2:** Instead of measuring the internal cohesion of a category to determine if it is "good", the scoring logic will measure **Cross-Category Collisions** (The Red Herring Matrix). The difficulty of a board will be quantified by counting how many times words from *different* categories cross a high similarity threshold, actively attempting to deceive the player.
